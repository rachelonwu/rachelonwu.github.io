Client-Side Routing Writeup

Routes

  • / — Home page route. This shows the list of chats and lets the user create a new chat.
  • /chat/:chatId — Dynamic chat route. The chatId part changes depending on which chat the user opens.
  • /chat/:chatId/digest — Dynamic chat digest route. This shows only the important messages from one specific chat.
  • /digest — Daily digest route. This shows important messages from across chats.

Design Decisions

I used client-side routing so the app feels like multiple pages while still working as one Vue app. The home route is for browsing chats, the chat route is for participating in a specific conversation, and the digest route is for reviewing important messages.

I made the chat page a dynamic route because each chat should have its own URL. This matches the structure of the app because every chat has its own identity and can eventually map to its own Graffiti channel.

I kept the daily digest as a separate route because it is a different mode from normal chatting. Instead of showing every message, it focuses only on messages that were marked as important.

I added a separate dynamic route for each chat’s digest because a chat digest should only summarize important messages from that one chat. The home digest is separate because it summarizes important messages across all chats.
